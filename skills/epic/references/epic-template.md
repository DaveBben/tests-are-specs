# Epic Template

Copy-paste starting point for each epic document. When multiple epics are
produced from an initiative, each epic gets its own file following this
format.

```markdown
Epic: [Short descriptive title]
Status: Proposed
Parent Initiative: [Initiative title or path, if applicable]

## Goal

[1-2 sentences describing the business or user outcome this epic delivers.
Not a feature list — an outcome. What becomes possible or stops being
painful once this exists?]

## Success Metric

[The specific, measurable outcome that defines success. Include a target
number or threshold and what it measures. Keep it to 1-2 metrics — this
is an epic, not a dashboard.]

## Why This Is [First/Second/Third/Last]

[Why this epic is sequenced where it is relative to the other epics in
the initiative. What does it depend on? What risk does it retire? What
does it unblock? If this is a standalone epic with no ordering context,
omit this section.]

## Rough Scope

[A paragraph describing what this epic covers. Specific enough that
someone reading it knows what work is included, but not so detailed
that it prescribes implementation. Think capabilities and behaviors,
not tasks.]

## Out of Scope

[What this epic explicitly does NOT cover. Name what someone might
assume is included but isn't. Reference which other epic handles it
if applicable. At least 1-2 specific exclusions.]

## Definition of Done

[A concrete sentence describing what "complete" looks like. The test:
could someone watch a demo and confirm this epic is done? Should
describe the end-to-end outcome, not a checklist of tasks.]
```
